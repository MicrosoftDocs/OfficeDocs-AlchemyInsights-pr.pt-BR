# <a name="map-a-sharepoint-library-to-a-network-drive"></a>Mapear uma biblioteca do SharePoint para uma unidade de rede

O mapeamento de uma biblioteca como uma unidade de rede é temporário e suportado apenas através do Internet Explorer. Ocasionalmente, você deve abrir o site do SharePoint no Internet Explorer e selecionar "permanecer conectado" para impedir que a sessão expire. Em vez disso, [sincronize os arquivos do SharePoint com o novo cliente](https://support.office.com/article/sync-sharepoint-files-with-the-new-onedrive-sync-client-6de9ede8-5b6e-4503-80b2-6190f3354a88) </a> de sincronização do onedrive, que fornece [arquivos sob demanda](https://support.office.com/article/learn-about-onedrive-files-on-demand-0e6860d3-d9f3-4971-b321-7092438fb38e). Acessar todos os seus arquivos no OneDrive sem usar espaço de armazenamento local.

Se você optar por mapear uma unidade em vez de[usar o novo cliente de sincronização do onedrive](https://support.office.com/article/sync-sharepoint-files-with-the-new-onedrive-sync-client-6de9ede8-5b6e-4503-80b2-6190f3354a88), certifique-se de seguir as etapas no artigo abaixo. 


**Como configurar e solucionar problemas de unidades de rede mapeadas**


- [Configurar e solucionar problemas de unidades de rede mapeadas](https://support.office.com/article/troubleshoot-mapped-network-drives-that-connect-to-sharepoint-online-ef399c67-4578-4c3a-adbe-0b489084eabe?ui=en-US&amp;rs=en-US&amp;ad=US).

Observação: para o Internet Explorer 10 com Windows 8 ou Windows 7 que receber "acesso negado" ou "o caminho não está acessível" ao mapear uma unidade, instale este hotfix para resolver esse problema. Ir para [erro ao abrir uma biblioteca de documentos do SharePoint no Windows Explorer ou mapear uma unidade de rede para a biblioteca após instalar o Internet Explorer 10](https://support.microsoft.com/help/2846960).
