# <a name="error-404-when-accessing-files-in-sharepoint-or-onedrive"></a>Erro 404 ao acessar arquivos no SharePoint ou no OneDrive

O <strong>erro 404 </strong>é recebido quando os usuários tentam acessar um site ou arquivo no SharePoint ou no onedrive. Isso geralmente é causado por um site ou arquivo ou grupo sendo renomeado, movido ou excluído.
Os usuários ocorrerão um erro 404 ao tentar acessar o conjunto de sites raiz e ele foi excluído.

Para resolver o erro 404 de um site que foi renomeado, movido ou excluído: para sites clássicos existentes no centro de administração clássico, confira <a href="https://docs.microsoft.com/en-us/sharepoint/restore-deleted-site-collection">restaurar um conjunto de sites excluído</a>.&nbsp;</li> <li>Para sites modernos (comunicação, grupo conectado ou outros sites) que existem na nova visualização do centro de administração, confira <a href="https://docs.microsoft.com/en-us/sharepoint/view-and-restore-deleted-sites-in-new-admin-center">exibir e restaurar sites excluídos no novo centro de administração do SharePoint. 

Para resolver o arquivo (ou outro item) que foi renomeado, movido ou excluído, vá para o site do SharePoint ou do OneDrive e exiba a lixeira do conteúdo do site, confira [restaurar itens na lixeira de um site do SharePoint](https://support.office.com/en-us/article/Restore-items-in-the-Recycle-Bin-of-a-SharePoint-site-6df466b6-55f2-4898-8d6e-c0dff851a0be).

 Se ainda não for possível localizar o item, você poderá pesquisar o log de auditoria se o log estiver habilitado </span>, confira <span style="mso-bidi-font-family: Calibri; mso-bidi-theme-font: minor-latin;"> <a style="box-sizing: border-box; orphans: 2; -webkit-text-stroke-width: 0px; word-spacing: 0px;" href="https://support.office.com/client/search-the-audit-log-in-the-office-365-security-compliance-center-0d4d0f35-390b-4518-800e-0c7ec95e946c"> <span style="color: #0067b8;">Pesquisar o log de auditoria no &amp; centro de conformidade de segurança do Office 365</span></a></span></p>
